# Modelgov v0.6.0 — Trustworthy audit trail and cost ledger

**Tag date:** 2026-07-01

Remediation release from the 2026-07-01 principal-engineer codebase review.
Policy blocks on fallback re-evaluation are enforced, every rejection path is
audited, classifier spend is booked on blocks, and idempotency replays expire
after 7 days. All High and Medium review findings are closed; the flat,
file-config path remains the default.

## Behavior changes (read before upgrading)

Standard SemVer 0.x: these ship in a minor because clients can observe them.

1. **Fallback data-sensitivity blocks are enforced.** When the primary
   provider fails and the fallback model's provider is not approved for the
   feature's data class, the request returns `403 policy_blocked`
   (`data_sensitivity_not_permitted`). Previously the block was silently
   ignored — the failed primary was retried and the audit log falsely recorded
   `decision: "fallback"`. Handle this 403 wherever you handle provider-outage
   `502`s.
2. **Classifier spend is booked on rejected requests.** The input-safety
   classifier makes a real, billable provider call; that cost now lands in
   `used_usd` on every path where it was incurred — safety block, reservation
   failure, fallback top-up failure, provider failure — and the audit row
   carries it as `actual_cost_usd`. Booking never gates: a safety block stays
   `403 safety_blocked` even when the spend pushes a counter past its cap; the
   overshoot surfaces on the next request's policy gate. Features with strict
   injection/PII presets incur per-request classifier cost **even on blocked
   requests**. See [docs/failure-semantics.md](../docs/failure-semantics.md).
3. **Idempotency replays expire after 7 days** (configurable:
   `IDEMPOTENCY_COMPLETED_RETENTION_MS`). Replaying an older key re-executes
   the request instead of returning the cached result. Previously completed
   keys were retained (and the table grew) forever.
4. **Error envelopes:** hierarchical and streaming rejections now include
   `auditRequestId` like the flat path; hierarchical policy-block errors no
   longer report flat `budgetRemaining` (it was computed against zero flat
   usage and claimed full headroom while the node tree is the authority).

## Audit-trail integrity

- **Every rejection writes a `request_logs` row** — the fallback top-up
  failure and streaming reservation-failure paths previously wrote none. A
  dedicated integration suite (`rejection-audit.integration.test.ts`) pins the
  invariant per rejection path.
- **Settlement retry fix** — after a fallback top-up, a failed-then-retried
  settlement released only the original reservation, stranding the top-up
  portion in `reserved_usd` with its lease already deleted.

## One lifecycle, three composers

Failure semantics — the audit trio, incur-then-release ordering, the fallback
block check, provider execution with fallback — now live once in
`chat/lifecycle.ts` (+ `providerBudget.ts` strategies, `authz/scope.ts` shared
scope checks) and are composed by the flat, hierarchical, and streaming
handlers. A fix like "honor the fallback block" can no longer exist in one
path and be missing from another.

## Gates that tell the truth

- **Coverage measures all of `packages/api/src` + policy-engine** — the old
  19-file allow-list reported 95.7% while chat, db, and services went
  unmeasured. Honest thresholds (81 lines / 72 branches / 89 functions),
  ratchet-up only.
- **CI runs the policy regression suite** (`modelgov.policy-tests.yaml`).
- **Trivy scans every Docker build** (previously PRs only), pinned to the
  release commit SHA rather than a mutable tag.

## Production defaults

- `.env.production.example` ships `DATABASE_SSL=require` (was `disable`).
- LiteLLM and Presidio have healthchecks; the API waits on `service_healthy`
  for all five dependencies.
- Boot warns when OIDC is enabled without `OIDC_AUDIENCE`.

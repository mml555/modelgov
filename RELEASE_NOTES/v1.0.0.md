# Modelgov v1.0.0 — First stable release

**Tag date:** 2026-07-02

The first stable, public release of Modelgov under the MIT license. From this
version the compatibility guarantees in [docs/versioning.md](../docs/versioning.md)
are in effect: breaking changes to the **HTTP API**, **SDKs**, or **config
schema** require a new major version.

This release consolidates the production-readiness hardening and multi-tenant
isolation completed in the 2026-07-01 audit with the gateway extensions and
reproducible builds added since 0.6.0.

## Highlights

- **Multi-tenant isolation across every previously-shared surface** — budget
  nodes, admin API keys, idempotency keys, flat budget counters (incl.
  `global_monthly`), and the admin audit log are all scoped to the caller's
  tenant, with no cross-tenant existence oracle. API keys carry a privilege
  ceiling (a key-admin can't mint cross-tenant keys or grant control-plane
  permissions it doesn't hold).
- **A money path that doesn't leak or double-book** — client-aborted streams
  bill the tokens actually produced (capped at the reservation); settlement is
  idempotent under retry; hierarchical policy blocks are enforced with `403`.
- **Gateway extensions** — governed embeddings (`POST /v1/embeddings`), vision /
  multimodal chat, a grounding safety mode (citation-verified answers against
  caller-supplied `context`), and a `pii_scope` masking control.
- **Reproducible container builds** — runtime image dependencies resolve
  entirely from the workspace lockfile (`pnpm deploy`), so the same commit
  yields the same dependency tree.
- **Operability** — domain Prometheus metrics, request-id correlation across
  logs/errors/headers, a finished operator console (with a turnkey nginx
  image), strict config validation (a mistyped budget can never fail open), and
  a Helm NetworkPolicy locking the sidecars to the API pod.

## ⚠ Breaking (vs 0.6.0)

- **Config schema:** unknown/misspelled top-level or budget keys in
  `modelgov.yaml` are now rejected (previously ignored). Validate with
  `modelgov validate --production` before upgrading.
- **HTTP API:** `POST /v1/chat` success responses return `budgetRemaining: null`
  under hierarchical budgets (the node tree is the authority). The field is
  nullable in the OpenAPI spec.
- **Database:** additive migrations `0020`–`0022` add tenant dimensions to
  `idempotency_keys`, `budget_counters` (+ reservation leases), and
  `admin_audit_log`, changing their primary keys. Forward-only; existing rows
  default to the untenanted (`''`) partition.

## Deploy notes

- Run migrations on deploy (through `0022_admin_audit_log_tenant.sql`).
- Pin the image: `ghcr.io/mml555/modelgov/modelgov-api:v1.0.0` (or its digest).
- The flat, file-config path remains the default; hierarchical budgets and
  per-tenant policy are opt-in.

## Boundary (unchanged)

```text
Your app     → Who may use this feature? (RBAC, business rules)
Modelgov     → Is this AI call allowed? (budget, model, safety, routing)
```

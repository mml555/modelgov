# Modelgov v0.0.0 ‚Äî Pre-release baseline

First aligned semver tag for the pre-1.0 stage. All publishable surfaces ‚Äî
npm packages, PyPI `modelgov`, Helm `appVersion`, OpenAPI `info.version`,
and the GHCR API image tag ‚Äî are pinned at **0.0.0**.

## What this means

- **SemVer 0.x** applies: minor bumps may include breaking changes until 1.0.
- Prior tags `v0.5.0` and `v0.6.0` were internal milestones; **`v0.0.0` is the
  canonical starting line** for external consumers.
- Deploy with image tag `v0.0.0` (or `main` for bleeding edge).

## Surfaces at 0.0.0

| Surface | Location |
| --- | --- |
| npm monorepo root | `package.json` |
| `@modelgov/*` + `create-modelgov` | `packages/*/package.json` |
| PyPI | `packages/sdk-python/pyproject.toml`, `modelgov.__version__` |
| OpenAPI | `packages/api/openapi.json`, `OPENAPI_VERSION` |
| Helm | `deploy/helm/modelgov/Chart.yaml` (`appVersion`) |
| Container | `ghcr.io/.../modelgov-api:v0.0.0` |

## Migration from internal tags

If you pinned `v0.6.0` during development, move to `v0.0.0` (or track `main`).
No wire-format break is implied by the version reset ‚Äî only the semver label
changes.

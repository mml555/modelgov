# Ai-Guard v0.0.0 — Pre-release baseline

First aligned semver tag for the pre-1.0 stage. All publishable surfaces —
npm packages, PyPI `ai-guard-sdk`, Helm `appVersion`, OpenAPI `info.version`,
and the GHCR API image tag — are pinned at **0.0.0**.

## What this means

- **SemVer 0.x** applies: minor bumps may include breaking changes until 1.0.
- Prior tags `v0.5.0` and `v0.6.0` were internal milestones; **`v0.0.0` is the
  canonical starting line** for external consumers.
- Deploy with image tag `v0.0.0` (or `main` for bleeding edge).

## Surfaces at 0.0.0

| Surface | Location |
| --- | --- |
| npm monorepo root | `package.json` |
| `@ai-guard/*` + `create-ai-guard` | `packages/*/package.json` |
| PyPI | `packages/sdk-python/pyproject.toml`, `ai_guard.__version__` |
| OpenAPI | `packages/api/openapi.json`, `OPENAPI_VERSION` |
| Helm | `deploy/helm/ai-guard/Chart.yaml` (`appVersion`) |
| Container | `ghcr.io/.../ai-guard-api:v0.0.0` |

## Migration from internal tags

If you pinned `v0.6.0` during development, move to `v0.0.0` (or track `main`).
No wire-format break is implied by the version reset — only the semver label
changes.

# Generating a client from the OpenAPI spec

Ai-Guard ships an OpenAPI 3 document so you can generate a typed client in any
language instead of hand-rolling HTTP calls.

## Get the spec

- **Committed:** [`packages/api/openapi.json`](../packages/api/openapi.json)
- **Live:** `GET /openapi.json` from a running gateway

## Auth

Every non-health endpoint takes `Authorization: Bearer <AI_GUARD_API_KEY>`. The
generated client should attach it on each request (config below).

## TypeScript (`@hey-api/openapi-ts`)

```bash
npx @hey-api/openapi-ts -i packages/api/openapi.json -o src/ai-guard-client
```

Then set the bearer token via the generated client's config. For most apps the
first-party [`@ai-guard/sdk`](./sdk-typescript.md) is simpler (typed
`feature`/`userType` from your `ai-guard.yaml`, typed errors) — generate only if
you need a non-TS language or a custom transport.

## Other languages (`openapi-generator`)

```bash
openapi-generator-cli generate \
  -i packages/api/openapi.json -g python -o ./ai-guard-python
# swap -g for: go, java, csharp, ruby, rust, ...
```

For Python, prefer the maintained [`ai-guard-sdk`](../packages/sdk-python)
(streaming + typed errors) unless you specifically want a generated client.

## Notes

- The spec is regenerated by `pnpm --filter @ai-guard/api openapi:export` and
  committed, so it always matches the shipped API.
- Streaming (`stream: true`) is `text/event-stream` and isn't fully expressible
  in OpenAPI — use an SDK's streaming method or read `data:` frames directly.
